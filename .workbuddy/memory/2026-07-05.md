# 2026-07-05 工作日志

## JTLP 截屏录屏工具 — 从零搭建

**需求**: 用户彻底清除旧项目，从最小化闭环出发搭建全新截屏工具 JTLP。

**项目结构**:
- `src/JTLP/JTLP.csproj` — .NET 8 WinForms（独立发布，含 .NET 运行时）
- `src/JTLP/Program.cs` — 主窗体 + 托盘 + 截图/录屏 + 热键
- `src/JTLP/AppSettings.cs` — 配置模型（JSON 持久化）
- `src/JTLP/ScreenRecorder.cs` — FFmpeg 录屏器（自动分段 + 合并）
- `src/JTLP/SettingsForm.cs` — 设置窗口（ListBox 导航 + Panel 面板）
- `tools/ffmpeg/ffmpeg.exe` — FFmpeg 依赖
- `dist/` — 发布产物（JTLP.zip + 安装脚本）

**已实现功能**:
1. 全屏截图（Alt+1）→ 保存到「截图中转站」，通知 "JT right"
2. 录屏（Alt+2 开始 / Alt+3 停止）→ 保存到「录屏中转站」，通知 "LP start"/"LP ending"
3. 系统托盘常驻 + 右键菜单 + 左键打开设置
4. 设置页面：热键修改、保存路径、录制分辨率、自动分段、压缩质量、开机自启
5. 单实例校验（Mutex）
6. 热键冲突检测（MessageBox 提示）
7. 存储路径支持复制和打开文件夹
8. 剩余存储空间检测（<1GB 标红）
9. 自动分段（默认10分钟）+ 停止后自动合并
10. 压缩质量设置（高/中/低，不同 CRF/preset）
11. 录制分辨率设置（原始/1080p/720p/480p）
12. DPI 感知 + 多显示器支持
13. 独立发布（self-contained，无需安装 .NET）

**项目决策记录**:
- 技术栈: C# .NET 8 / WinForms（非 partial class）
- UI 布局: ListBox 导航 + Panel 面板（左右叠代，不用 TabControl）
- 启动流程: 启动 → 弹设置页面 → 最小化到托盘或退出
- 截图行为: 自动保存 + 通知
- 录屏行为: FFmpeg gdigrab + 自动分段 + 停止后合并
- 配置持久化: JSON 文件（exe同级 jtlp-settings.json）
- 分发方式: ZIP + PowerShell 安装脚本

**编译**: 0 错误 0 警告

## 关键 Bug 修复

1. **SettingsForm 类名冲突** — `ImageFormat` 重命名为 `ScreenshotFormat`
2. **SettingsForm 空面板** — `OnLoad` 中先触发面板创建再加载设置
3. **录屏无反应** — FFmpeg 路径向上遍历查找 + 异步检查存活
4. **UI 线程阻塞** — 去掉 `Thread.Sleep`，改用 Timer 异步检查
5. **单实例校验** — `Mutex` 防止重复启动
